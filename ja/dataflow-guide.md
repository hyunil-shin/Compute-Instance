## Data & Analytics > DataFlow > コンソール使用ガイド

DataFlowは以下の順序で使用できます。

* サービス有効化
    1. プロジェクトを作成します。
    2. 目的のプロジェクトを選択します。
    3. DataFlowサービスを有効化します。
* フロー実行
    1. フローを作成します。
    2. 必要なノードを追加し、設定値を入力して各ノードの動作方式を定義します。
    3. ノード接続を通じてノードの動作順序を決定してフローを完成させます。
    4. フローを実行します。
    5. ログ情報を確認してフローが正常に実行されたかどうかを確認します。

## 管理

フローメタデータ情報を照会および管理するページです。
**Data & Analytics > DataFlow > 管理**をクリックします。

### 検索

指定された基準でフローを検索します。

* フロー名を基準に検索すると、名前に検索語が含まれたフローを検索します。

### フィルター

指定された条件でフローを検索します。

* フロー状態値に応じたフィルタリングオプションを提供します。

### フロー一覧

検索結果をテーブル形式でフローを表示します。

* 簡潔なフロームタデータと現在のフロー実行状態を表示します。
* 最終変更日を基準にソートします。
* フローを選択してフロー変更、フロー詳細表示、フロー開始などの管理機能を使用できます。
* フィルター条件機能を通じて特定状態のフローのみを照会できます。
* 1回照会されたフローは、**更新**をクリックして検索結果を更新する必要があります。
* 1ページあたり12個のフローを照会でき、**前へ**および**次へ**をクリックしてページを移動できます。

#### フロー状態情報

| フロー実行状態                                         | 説明 |
|---------------------------------------------------| --- |
| START\_FAILED  | フロー実行リクエストが失敗しました。 |
| QUOTA\_EXCEEDED | フロー実行のためのリソースが不足して実行に失敗しました。 |
| STARTING       | フロー実行のためのリソースを確保中です。 |
| PREPARING      | フロー実行準備が完了しました。 |
| RUNNING        | フローが実行中です。 |
| ERROR              | フロー実行過程で通信障害や認証不可などによってエラーが発生しました。継続的に<b>ERROR</b>が発生する場合は、**カスタマーサポート > お問い合わせ**からお問い合わせください。 |
| STOP\_FAILED   | フロー終了リクエストが失敗しました。 |
| STOPPED        | フローが終了しました。 |
| DRAINING       | フローがドレイニング中です。 |
| UNKNOWN            | フロー実行過程で不明な原因によってエラーが発生しました。継続的に<b>UNKNOWN</b>が発生する場合は、**カスタマーサポート > お問い合わせ**からお問い合わせください。 |

#### フロー状態変更通知メール
* 通知対象フロー状態に変更されたときに通知メールを受信できます。
* 通知対象フロー状態
    * RUNNING
    * ERROR
    * STOPPED
* デフォルト受信対象
    * 使用中のDataFlowサービスが有効化されたプロジェクトの**DataFlow ADMIN**ロールを持つメンバー


### フロー作成

フローを定義するメタデータを作成します。

* フロー識別のために名前と説明を追加してフロームタデータを作成します。
* フロー名は他のフローと重複が可能です。
* フロー目的に応じた実行モードを選択します。
* フロープランテンプレートを指定して、ユーザーが希望する機能のフローを簡単に読み込むことができます。
* フローを実行するためのインスタンスタイプを設定できます。 

### フロー変更

フローのメタデータを修正します。

* 既存のフロー名と説明を修正してフロームタデータに反映します。
* フロープランテンプレートは指定できません。
* フロー実行中でもフロー変更が可能です。
* フローを実行するためのインスタンスタイプを変更できます。
    * ただし、変更されたインスタンスタイプは次のフロー実行から適用されます。

### フロー複製

既存のフロー定義を持って新しいメタデータを作成します。

* 既存フローの名前に`_copy`文字列が追加された新しいメタデータを作成します。
* 既存フローが持つフロー論理をそのままコピーします。
* 実行中のフローも複製が可能です。
* 実行中のフローは停止状態で複製します。
* フローの現在の保存状態が一時保存の場合、既存フローの最後の保存バージョンは複製しません。
* スケジューラーが登録されたフローを複製しても、複製されたフローにはスケジューラーが登録されません。
* 複製されたフローは既存フローと完全に別個のフローです。

### フロー削除

フロームタデータを削除します。

* フロームタデータを完全に削除します。
* 削除されたフローは復旧できません。
* 実行中のフローは削除できません。

### その他 - フロー開始
停止状態のフローを開始します。

* 各フローは同時に1つだけ実行できます。
* 一時保存されたフローは最後に保存されたバージョンで開始します。
* 一度も保存せず一時保存だけしたフローは開始できません。
* フローは必ず1回以上保存してから開始できます。
* フローがスケジューラーによって実行中でも、ユーザーによって開始されたフローと同じようにフローを開始できません。

### その他 - フロー終了
* 実行準備中または実行中、ドレイニング中のフローを終了できます。
* 残余イベントは処理せず終了します。

### その他 - フロー ドレイニング後終了
* 実行中のフローをドレイニング後に終了できます。
* ドレイニングはフローの残余イベントを処理することを意味します。
* タイムアウト時間を超過した場合、ドレイニングが終わらなくても終了します。
* タイムアウト時間が残っている状態でドレイニングが終了した場合、すぐに終了します。
* ドレイニング中のフローはフロー終了を通じてすぐに終了できます。
