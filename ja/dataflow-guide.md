## Data & Analytics > DataFlow > コンソール使用ガイド

DataFlowは以下の順序で使用できます。

* サービス有効化
    1. プロジェクトを作成します。
    2. 目的のプロジェクトを選択します。
    3. DataFlowサービスを有効化します。
* フロー実行
    1. フローを作成します。
    2. 必要なノードを追加し、設定値を入力して各ノードの動作方式を定義します。
    3. ノード接続を通じてノードの動作順序を決定してフローを完成させます。
    4. フローを実行します。
    5. ログ情報を確認してフローが正常に実行されたかを確認します。

## 管理

フロータイプデータ情報を照会および管理するページです。
**Data & Analytics > DataFlow > 管理**をクリックします。

![management_main.png](http://static.toastoven.net/prod_dataflow/ko/console_user_guide/management_main_2025_06.png)

### 検索

与えられた基準でフローを検索します。

* フロー名を基準に検索すると、名前に検索語が含まれるフローを検索します。

### フィルター

与えられた条件でフローを検索します。

* フロー状態値に基づくフィルタリングオプションを提供します。

### フロー一覧

照会結果のフローをテーブル形式で表示します。

* 簡単なフロータイプデータとフローの現在の実行状態を表示します。
* 最終更新日を基準にソートします。
* フローを選択してフロー変更、フロー詳細表示、フロー開始などの管理機能を使用できます。
* フィルター条件機能を通じて特定状態のフローのみ照会できます。
* 一度照会されたフローは**更新**を押して照会結果を更新する必要があります。
* 1ページあたり12個のフローを照会でき、**前へ**および**次へ**をクリックしてページを移動できます。

#### フロー実行状態情報

| フロー実行状態                                         | 説明 |
|---------------------------------------------------| --- |
| START\_FAILED  | フロー実行リクエストに失敗しました。 |
| QUOTA\_EXCEEDED | フロー実行のためのリソースが不足して実行に失敗しました。 |
| STARTING       | フロー実行のためのリソースを確保中です。 |
| PREPARING      | フロー実行準備が完了しました。 |
| RUNNING        | フローが実行中です。 |
| ERROR              | フロー実行過程で通信障害や認証不可などによってエラーが発生しました。継続して<b>ERROR</b>が発生する場合は**カスタマーサポート > お問い合わせ**からお問い合わせください。 |
| STOP\_FAILED   | フロー終了リクエストに失敗しました。 |
| STOPPED        | フローが終了しました。 |
| DRAINING       | フローがドレイニング中です。 |
| UNKNOWN            | フロー実行過程で不明な原因によってエラーが発生しました。継続して<b>UNKNOWN</b>が発生する場合は**カスタマーサポート > お問い合わせ**からお問い合わせください。 |

#### フロー状態変更通知メール
* 通知対象フロー状態に変更されたときに通知メールを受信できます。
* 通知対象フロー状態
    * RUNNING
    * ERROR
    * STOPPED
* デフォルト受信対象
    * 使用中のDataFlowサービスが有効化されたプロジェクトの**DataFlow ADMIN**ロールを持つメンバー


### フロー作成

フローを定義するタイプデータを作成します。

* フロー識別のために名前と説明を追加してフロータイプデータを作成します。
* フロー名は他のフローとの重複が可能です。
* フロー目的に応じた実行モードを選択します。
* フロータンプレートを指定して、ユーザーが希望する機能のフローを簡単に読み込むことができます。
* フロー実行のためのインスタンスタイプを設定できます。 

### フロー変更

フローのタイプデータを変更します。

* 既存のフロー名と説明を変更してフロータイプデータに反映します。
* フロータンプレートは指定できません。
* フロー実行中でもフロー変更が可能です。
* フロー実行のためのインスタンスタイプを変更できます。
    * ただし、変更されたインスタンスタイプは次のフロー実行から適用されます。

### フロー複製

既存のフロー定義を持つ新しいタイプデータを作成します。

* 既存フローの名前に`_copy`文句が追加された新しいタイプデータを作成します。
* 既存フローが持つフロー論理をそのままコピーします。
* 実行中のフローもコピーが可能です。
* 実行中のフローは停止状態でコピーします。
* フローの現在保存状態が仮保存の場合、既存フローの最後の保存バージョンはコピーしません。
* スケジューラーが登録されたフローをコピーしても、コピーされたフローはスケジューラーが登録されません。
* コピーされたフローは既存フローと完全に別個のフローです。

### フロー削除

フロータイプデータを削除します。

* フロータイプデータを完全に削除します。
* 削除されたフローは復旧できません。
* 実行中のフローは削除できません。

### その他 - フロー開始
停止状態のフローを開始します。

* 各フローは同時に1つだけ実行できます。
* 仮保存したフローは最後に保存したバージョンで開始します。
* 一度も保存せず仮保存のみしたフローは開始できません。
* フローは必ず1回以上保存してから開始できます。
* フローがスケジューラーによって実行中でも、ユーザーによって開始されたフローと同じようにフローを開始できません。

### その他 - フロー終了
* 実行準備中または実行中、ドレイニング中のフローを終了できます。
* 残余イベントは処理せずに終了します。

### その他 - フロードレイニング後終了
* 実行中のフローをドレイニング後に終了できます。
* ドレイニングはフローの残余イベントを処理することを意味します。
* タイムアウト時間を超えた場合、ドレイニングが終了していなくても終了します。
* タイムアウト時間が残っている状態でドレイニングが終了する場合、すぐに終了します。
* ドレイニング中のフローはフロー終了を通じてすぐに終了できます。
