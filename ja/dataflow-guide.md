## Data & Analytics > DataFlow > コンソール使用ガイド

DataFlowは次の順序で使用できます。

* サービス有効化
    1. プロジェクトを作成します。
    2. 目的のプロジェクトを選択します。
    3. DataFlowサービスを有効化します。
* フロー実行
    1. フローを作成します。
    2. 必要なノードを追加し、設定値を入力して各ノードの動作方式を定義します。
    3. ノード接続を通じてノードの動作順序を決定してフローを完成させます。
    4. フローを実行します。
    5. ログ情報を確認してフローが正常に実行されたか確認します。

## 管理

フロー メタデータ情報を照会して管理するページです。
**Data & Analytics > DataFlow > 管理**をクリックします。

![management_main.png](http://static.toastoven.net/prod_dataflow/ko/console_user_guide/management_main_2025_06.png)

### 検索

指定された基準でフローを検索します。

* フロー名を基準に検索するとフロー名に検索語が含まれたフローを検索します。

### フィルター

指定された条件でフローを検索します。

* フローステータス値に応じたフィルタリングオプションを提供します。

### フロー一覧

照会結果をテーブル形式でフローを表示します。

* シンプルなフロー メタデータと現在のフロー実行ステータスを表示します。
* 最終更新日を基準にソートします。
* フローを選択してフロー変更、フロー詳細表示、フロー開始などの管理機能を使用できます。
* フィルター条件機能を通じて特定ステータスのフローのみ照会できます。
* 1回照会されたフローは**最新情報に更新**を押して照会結果を更新する必要があります。
* 1ページあたり12個のフローを照会し、**前へ**および**次へ**をクリックしてページを移動できます。

#### フロー実行ステータス情報

| フロー実行ステータス | 説明 |
|---------------------------------------------------| --- |
| START\_FAILED  | フロー実行リクエストに失敗しました。 |
| QUOTA\_EXCEEDED | フロー実行のためのリソースが不足して実行に失敗しました。 |
| STARTING       | フロー実行のためのリソースを確保中です。 |
| PREPARING      | フロー実行準備が完了しました。 |
| RUNNING        | フローが実行中です。 |
| ERROR              | フロー実行過程で通信障害や認証不可などが原因でエラーが発生しました。継続的に<b>ERROR</b>が発生する場合は、**カスタマーサポート > お問い合わせ**でお問い合わせください。 |
| STOP\_FAILED   | フロー終了リクエストに失敗しました。 |
| STOPPED        | フローが終了しました。 |
| DRAINING       | フローがドレイニング中です。 |
| UNKNOWN            | フロー実行過程で未知の原因によるエラーが発生しました。継続的に<b>UNKNOWN</b>が発生する場合は、**カスタマーサポート > お問い合わせ**でお問い合わせください。 |

#### フロー ステータス変更通知メール
* 通知対象フロー ステータスに変更されたときに通知メールを受信できます。
* 通知対象フロー ステータス
    * RUNNING
    * ERROR
    * STOPPED
* デフォルト受信対象
    * 使用中のDataFlowサービスが有効化されたプロジェクトの**DataFlow ADMIN**ロールを持つメンバー


### フロー作成

フローを定義するメタデータを作成します。

* フロー識別のため、名前と説明を追加してフロー メタデータを作成します。
* フロー名は他のフローとの重複が可能です。
* フロー目的に応じた実行モードを選択します。
* フローテンプレートを指定してユーザーが目的の機能のフローを簡単に読み込むことができます。
* フローを実行するためのインスタンスタイプを設定できます。 

### フロー変更

フロー メタデータを修正します。

* 既存のフロー名と説明を修正してフロー メタデータに反映します。
* フローテンプレートは指定できません。
* フロー実行中でもフロー変更が可能です。
* フローを実行するためのインスタンスタイプを変更できます。
    * ただし、変更されたインスタンスタイプは次のフロー実行から適用されます。

### フロー複製

既存のフロー定義を使用して新しいメタデータを作成します。

* 既存フローの名前に`_copy`文字列が追加された新しいメタデータを作成します。
* 既存フローが持つフロー ロジックをそのままコピーします。
* 実行中のフローも複製が可能です。
* 実行中のフローは停止状態で複製します。
* フローの現在の保存状態が仮保存の場合、既存フローの最後の保存バージョンはコピーしません。
* スケジューラーが登録されたフローを複製しても、複製したフローはスケジューラーが登録されません。
* 複製したフローは既存フローと完全に別個のフローです。

### フロー削除

フロー メタデータを削除します。

* フロー メタデータを完全に削除します。
* 削除したフローは復元できません。
* 実行中のフローは削除できません。

### その他 - フロー開始
停止状態のフローを開始します。

* 各フローは同時に1つだけ実行できます。
* 仮保存したフローは最後に保存したバージョンで開始します。
* 1回も保存せず仮保存のみ行ったフローは開始できません。
* フローは必ず1回以上保存してから開始できます。
* フローがスケジューラーによって実行中でも、ユーザーが開始したフローと同様にフローを開始できません。

### その他 - フロー終了
* 実行準備中または実行中、ドレイニング中のフローを終了できます。
* 残留イベントは処理せずに終了します。

### その他 - フロー ドレイニング後終了
* 実行中のフローをドレイニング後に終了できます。
* ドレイニングはフローの残留イベントを処理することを意味します。
* タイムアウト時間を超える場合、ドレイニングが終了していなくても終了します。
* タイムアウト時間が残っている状態でドレイニングが終了する場合は即座に終了します。
* ドレイニング中のフローはフロー終了を通じてすぐに終了できます。
